## 1. 设计目标

### 1.1 背景

当前前端界面与产品设计预期不一致，影响产品测试和用户体验。本次重写目标是建立一套专业、克制、适合白天交易场景的界面设计体系。

### 1.2 核心目标

- 建立统一的极简终端风 (Pro Light) 设计体系
- 优化交易页面布局，提升操作效率
- 简化页面结构（合并仓位到交易页）
- 确保数据展示的清晰性和专业感

---

## 2. 设计系统

### 2.1 设计理念

**极简终端风 (Pro Light)**

专业交易员在白天办公时更倾向于明亮模式。这套方案干净、克制，凸显数据的严谨性，避免视觉疲劳。

### 2.2 配色方案

| 用途 | 名称 | Hex | 说明 |
|------|------|-----|------|
| 主背景 | 珍珠灰 | `#F8F9FA` | 略带灰度的白，避免纯白眩光 |
| 面板/卡片 | 纯净白 | `#FFFFFF` | 承载核心交易数据，干净利落 |
| 品牌主色 | 曜石黑 | `#0F172A` | 极致深色作为品牌色，展现克制与专业 |
| 点缀色 | 科技青 | `#0EA5E9` | 高亮链接、Hover效果，带来Web3气息 |
| 上涨/买入 | 森林绿 | `#059669` | 浅色背景下对比度更高的深绿色 |
| 下跌/卖出 | 砖红色 | `#DC2626` | 沉稳红色，确保白色面板上的易读性 |
| 主要文字 | 碳黑色 | `#1E293B` | 极高对比度，适合密集数据排版 |
| 次要文字 | 辅助灰 | `#64748B` | 标签、说明文字 |

### 2.3 字体规范

- **主字体**：系统字体栈 `-apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif`
- **等宽字体**：`'SF Mono', Monaco, monospace`（用于价格、数量等数据）
- **标题字号**：24px（页面标题）、16px（面板标题）、13px（标签）
- **正文字号**：15px（主要数据）、14px（正文）、13px（次要信息）、12px（辅助信息）
- **字重**：400（常规）、500（中等）、600（半粗）、700（粗体）

### 2.4 间距系统

- **页面内边距**：24px
- **面板内边距**：16px-24px
- **卡片间距**：12px-16px
- **元素间距**：8px-16px
- **圆角**：8px（面板/卡片）、6px（按钮/输入框）、50%（图标）

### 2.5 阴影规范

- **面板阴影**：`0 1px 3px rgba(0,0,0,0.04)`
- **悬浮阴影**：`0 2px 8px rgba(0,0,0,0.08)`
- **Hover阴影**：`0 4px 12px rgba(0,0,0,0.1)`

### 3.2 交易页面

#### 布局结构

**终端式左右分栏**：左侧主区域（约70%）+ 右侧交易区（固定360px）

```
+--------------------------- 左侧主区域 (约70%) ---------------------------+----------------- 右侧交易区 (固定360px) ------------------+
|                                                                        |                                                      |
|  [市场统计栏] 标记价格 | 指数价格 | 24h涨跌 | 24h最高 | 24h成交量        |   [交易表单]                                         |
|                                                                        |   - 多空切换标签                                     |
|  [图表区域]                                                            |   - 保证金输入（可用余额显示）                       |
|  ┌──────────────────────────────────────────────────────────────┐    |   - 杠杆滑块（1x-20x）                               |
|  │  [工具栏: 15m | 1H | 4H | 1D]                    BTC/USD      │    |   - 开多/开空按钮                                    |
|  │                                                              │    |   - 预估订单摘要                                   │
|  │   ┌─────────────┐                                           │    |     * 仓位大小、开仓价格、清算价格、手续费         │
|  │   │ BTC/USD     │   TradingView K线图                       │    |                                                      │
|  │   │ 67,245.50   │                                           │    |                                                      │
|  │   │ +2.34% ↑    │                                           │    |   [当前仓位面板]                                   │
|  │   └─────────────┘                                           │    |   - 仓位列表                                       │
|  │                                                              │    |   - 平仓按钮                                       │
|  └──────────────────────────────────────────────────────────────┘    |                                                      |
|                                                                        |                                                      |
|  [历史记录面板 - 固定高度280px]                                        |                                                      |
|  [最近交易 | 委托订单 | 资金流水]                                      |                                                      |
|  表格：时间 | 类型 | 数量 | 交易对 | 状态                              |                                                      |
|                                                                        |                                                      |
+------------------------------------------------------------------------+------------------------------------------------------+
```

**布局说明：**
- 左侧主区域：市场统计栏 + 图表区域（自适应高度）+ 历史记录面板（固定280px）
- 右侧交易区：交易表单（上部）+ 当前仓位面板（下部，自适应高度）
- 两区域之间的分隔线为 1px solid `#E2E8F0`

#### 左侧主区域

**1. 市场统计栏**
- 5列等宽布局
- 字段：标记价格、指数价格、24h涨跌、24h最高、24h成交量
- 数字使用等宽字体

**2. 图表区域**
- 最小高度：380px
- 工具栏：时间周期选择（15m/1H/4H/1D）+ 交易对名称
- 左上角悬浮价格卡片：
  - **位置**：固定于图表区域左上角（position: absolute; top: 16px; left: 16px）
  - **内容**：交易对名称（如 BTC/USD）、当前标记价格、24h涨跌幅
  - **数据更新**：跟随 WebSocket 实时推送更新，无动画过渡
  - **背景**：半透明白色背景（rgba(255,255,255,0.95)）+ 轻微阴影，确保在图表上方可读
  - **交互**：不可点击，纯信息展示

**3. 历史记录面板**
- 固定高度：280px
- 标签切换：最近交易 / 委托订单 / 资金流水
- 表格列：时间、类型、数量、交易对、状态

#### 右侧交易区

**1. 交易表单**
- 多空切换标签（绿色/红色视觉区分）
- 保证金输入框（带USDC后缀、显示可用余额）
- 杠杆滑块（1x-20x，视觉刻度）
- 开多/开空按钮（主色调填充）
- **预估订单摘要**：根据用户输入的保证金和杠杆实时计算
  - 仓位大小 = 保证金 × 杠杆
  - 开仓价格 ≈ 当前标记价格
  - 清算价格：参考 `docs/ARCHITECTURE.md` 第6.6节仓位模块的清算价格公式
    - 多头清算价 = 开仓价 × (1 - 初始保证金率 / 维持保证金率)
    - 空头清算价 = 开仓价 × (1 + 初始保证金率 / 维持保证金率)
    - MVP阶段使用简化公式：清算价 = 开仓价 × (1 ± 1/杠杆 × 0.9)
  - 手续费 = 仓位大小 × 手续费率
  - **手续费率**：从后端配置获取（API: `GET /api/config`），默认 0.05%，展示时保留4位小数
  - 数据随输入变化实时更新，无延迟（使用 React useMemo 计算）

**2. 当前仓位面板**
- 显示当前所有持仓
- 每个仓位卡片包含：
  - 交易对 + 方向标签（做多/做空）
  - 6个关键数据：仓位大小、未实现盈亏、开仓价格、标记价格、杠杆、清算价格
  - 平仓按钮

**纯列表式**，简洁清晰

## 6. 交互规范

### 6.1 状态反馈

- **Loading**：按钮禁用 + Spinner
- **成功**：绿色Toast提示
- **失败**：红色Toast提示 + 错误详情
- **确认**：模态框确认（如平仓、大额提现）

### 6.2 动画规范

- **过渡时间**：150ms-300ms
- **缓动函数**：ease-out
- **常见动画**：
  - Hover：背景色过渡
  - 面板展开：高度过渡
  - 模态框：淡入 + 缩放
  - Toast：滑入滑出

### 6.3 数据刷新策略

**价格数据**
- 来源：WebSocket 实时推送（Socket.IO）
- 机制：后端每2秒推送一次最新标记价格，前端监听 `market:price:update` 事件更新UI
- 展示：所有价格字段同步更新，无闪烁；若2秒内无数据推送，显示"连接中..."
- 断线重连：WebSocket断开时自动重连，最多3次，间隔1s/2s/4s

**余额/仓位数据**
- 刷新触发条件：
  1. **页面聚焦刷新**：监听 `visibilitychange` 事件，当页面从后台切换到前台时自动刷新
  2. **操作后刷新**：用户完成充值、提现、开平仓操作后，延迟 500ms 刷新（等待链上确认）
  3. **定时刷新**：每 30 秒自动刷新一次（作为兜底）
- 加载状态：显示骨架屏，不显示 Loading 遮罩

**历史记录**
- 首次加载：进入页面时加载第 1 页（20条）
- 分页加载：点击分页按钮时加载对应页
- 刷新：切换筛选条件时重新加载第 1 页
- 移动端：支持下拉刷新手势

## 7. 技术实现要点

### 7.1 技术栈

- **框架**：Next.js 15 + React 19
- **状态管理**：Zustand
- **样式**：Tailwind CSS
- **组件库**：shadcn/ui
- **图表**：TradingView Lightweight Charts
- **钱包**：Reown AppKit + Wagmi v2
- **数据获取**：TanStack Query

### 7.2 性能优化

**图表懒加载**
- 实现方式：使用 Next.js 动态导入 `next/dynamic`，配合 `ssr: false` 禁用服务端渲染
- 触发时机：
  1. 用户点击导航进入交易页时立即开始加载（组件级懒加载）
  2. 若用户直接访问首页（/），则延迟 500ms 后预加载图表组件
- 占位符：加载前显示骨架屏（灰色矩形脉冲动画，高度380px）
- 加载超时：10秒后若未加载成功，显示"加载失败，点击重试"

**历史记录分页 vs 虚拟滚动**
- **交易页历史面板**（固定高度280px）：
  - 采用分页加载，每页5条（刚好填满面板）
  - 不启用虚拟滚动，因数据量小且固定
- **历史记录页面**（全页表格）：
  - 采用服务端分页，每页20条
  - 当单页数据超过50条时（特殊情况），启用 `@tanstack/react-virtual` 虚拟滚动
  - 行高：固定52px

**图片/图标优化**
- 使用 SVG 图标（Lucide React），无需额外优化
- 资产图标使用 CSS 首字母生成，无需图片资源

**路由预加载**
- 实现方式：Next.js 自动路由预加载
- 额外配置：鼠标悬停在导航链接上时预加载对应页面数据（TanStack Query prefetch）

### 7.3 无障碍

- 支持键盘导航
- ARIA标签
- 高对比度模式适配
- 字体大小可调
